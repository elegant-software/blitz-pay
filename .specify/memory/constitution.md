# Project Constitution

## Purpose
Describe the product goals and user outcomes.

## Engineering Principles
- Keep changes small and reviewable.
- Prefer clear, testable behavior.
- Maintain backward compatibility unless explicitly agreed.

## Quality Standards
- Add tests for behavior changes.
- Keep documentation aligned with implementation.
- Avoid unrelated refactors in feature work.

## Governance
- Every new spec should reference these principles.
- Plans and tasks should call out trade-offs and risks.
