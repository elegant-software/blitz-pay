# GitSpecKit Workspace

This project has been prepared for Spec-Driven Development with GitSpecKit-style artifacts.

## What was initialized

- `.specify/memory/constitution.md` for project principles.
- `.specify/templates/` with starter templates for spec, plan, and tasks.
- `.specify/specs/` as the root for feature specs.

## Suggested next steps

1. Define your constitution in `.specify/memory/constitution.md`.
2. Create your first feature folder in `.specify/specs/` (for example: `001-your-feature`).
3. Fill in the templates to draft `spec.md`, `plan.md`, and `tasks.md`.
